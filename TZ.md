**Новые фичи**
1. Добавить авторизацию и регистрацию, она будет необходима по сути только для администратора, чтобы он мог входить в админ панель и выполнять все неоюходимые действия. Авторизация и регистрация должна быть с выводом ошибок валидации. По дизайну будет в том же стиле, что и основная страница

2. Разработать админ панель. Там на данный момент будут 2 вкладки:
    1) Вкладка "Проекты". Там будут все существующие проекты из лендинга. Админ сможет создавать новый проект, редактировать и удалять старые и новые проекты.
    Поля в форме: 
    a. Вставьте фотографию проекта. Ограничений по размеру фото не будет
    b. Название проекта
    c. В select выбор кол-ва этажей и в input метраж проекта
    d. Ключевые особенности проекта. Сейчас в карточках это например "4 спальни", "2 санузла", "Кабинет" и тд
    Далее  будут поля для заполнения модального окна "Планировки":
    a. Название проекта
    b. 4 input: Площадь, кол-во этажей, кол-во спален, кол-во санузлов
    c. При указании кол-ва этажей ниже автоматически появляются поля для прекрепления фотографий планировой под каждый этаж

    2) Вкладка заявки. Там в формате таблицы будут отображаться заявки от пользователей сайта, которые отправили заявку. Должны отображаться имя, номер телефона и если заявка отправлена по кнопке "Стоимость по запросу", то проект, с которого была отправлена заявка

    4) Создать в админ панели вкладку "Портфолио", где во-первых будут размещены существующие фотографии из блока порфтолио, которые можно удалить и во-вторых можно будет в этот блок добавлять свои фотографии, которые будут отображаться на лендинге
    5) При добавлении фотографии этажа в создании проекта в админ панели отображается следующая ошибка: The PUT method is not supported for route api/projects/1. Supported methods: POST, DELETE.
